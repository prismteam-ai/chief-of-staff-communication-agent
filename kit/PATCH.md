## Naming record (`build-ai-agents` / `foundation-agent-naming`)

- **Collision-checked:** `pidgeot` does not exist among the kit's 33 current agent names
  (`abra`, `alakazam`, `arceus`, `ash`, `audino`, `braviary`, `castform`, `chatot`, `conkeldurr`,
  `delibird`, `ditto`, `donphan`, `eevee`, `espeon`, `hoothoot`, `hypno`, `klefki`, `linoone`,
  `lucario`, `machamp`, `meowth`, `metagross`, `noctowl`, `oracle`, `oranguru`, `pelipper`,
  `porygon`, `regigigas`, `slowking`, `smeargle`, `sylveon`, `wigglytuff`, `xatu`).
- **Why Pidgeot:** Pidgeot is a bird Pokémon whose signature trait is speed and reach — carrying
  messages/riders swiftly across long distances. That maps directly onto this agent's role: the
  fast, always-available carrier between the human (in Cursor) and the Chief of Staff system —
  retrieving context, surfacing recommendations/drafts, and carrying the human's approval back to
  the hosted API to actually send. Plugin subagent name: plain `pidgeot` (kebab-case, matches
  `agents/pidgeot.md`'s filename).
- **Runtime identifier convention:** per `foundation-agent-naming.md`, a matching runtime would use
  `pidgeot-agent` as the identifier prefix (`LANGSMITH_PROJECT=pidgeot-agent`, `keyPrefix:
  'pidgeot-agent'`, etc). This particular agent is a Cursor MCP server, not an Asana-triggered
  Chat-SDK Lambda runtime (no LangSmith/AgentCore Memory wiring applies to it) — the identifier
  convention is recorded here for consistency and for any future runtime companion, not because
  this MCP server currently emits LangSmith telemetry.

## Apply steps (operator, in a `soofi-xyz-team-kit` checkout)

```bash
cd /path/to/soofi-xyz-team-kit
git checkout -b add-pidgeot-agent

# 1. Copy the new agent + skill (and pre-generated sync outputs) into place
cp /path/to/chief-of-staff-communication-agent/kit/agents/pidgeot.md agents/pidgeot.md
cp /path/to/chief-of-staff-communication-agent/kit/agents-copilot/pidgeot.agent.md agents-copilot/pidgeot.agent.md
cp /path/to/chief-of-staff-communication-agent/kit/.codex/agents/pidgeot.toml .codex/agents/pidgeot.toml
mkdir -p skills/use-pidgeot/reference
cp /path/to/chief-of-staff-communication-agent/kit/skills/use-pidgeot/SKILL.md skills/use-pidgeot/SKILL.md
cp /path/to/chief-of-staff-communication-agent/kit/skills/use-pidgeot/reference/*.md skills/use-pidgeot/reference/

# 2. Re-run sync to confirm the pre-generated copies are still correct (no-op if unchanged)
scripts/sync-copilot-agents.sh sync
scripts/sync-codex-agents.sh sync

# 3. Merge the mcp.json entry (kit/../mcp.json in this repo) into the kit's root mcp.json
#    under mcpServers — do NOT overwrite the existing "elephant" entry, add "pidgeot" alongside it.

# 4. Bump version 0.27.1 -> 0.28.0 (new-agent addition = minor bump) in all four places:
#    .cursor-plugin/plugin.json, .codex-plugin/plugin.json, plugin.json,
#    .github/plugin/marketplace.json (both "metadata".version and plugins[0].version)

# 5. Add the README roster row (see below) to the Agents table in README.md

# 6. Validate
scripts/validate-plugin.sh

# 7. Local Cursor smoke test
scripts/local-cursor-plugin.sh install
# Reload Cursor, Settings > Plugins, confirm soofi-xyz-team-kit-local, run:
#   /pidgeot Reply with exactly: ok
scripts/local-cursor-plugin.sh remove   # before opening the PR

git add agents/pidgeot.md agents-copilot/pidgeot.agent.md .codex/agents/pidgeot.toml \
  skills/use-pidgeot/ mcp.json .cursor-plugin/plugin.json .codex-plugin/plugin.json \
  plugin.json .github/plugin/marketplace.json README.md
git commit -m "feat(agents): add pidgeot — Chief of Staff communication agent for Cursor"
```

## `mcp.json` entry to merge

See `../mcp.json` in this repo (`chief-of-staff-communication-agent/mcp.json`) — the exact block,
reproduced here for convenience:

```jsonc
{
  "mcpServers": {
    "pidgeot": {
      "command": "bash",
      "args": [
        "-c",
        "exec npx -y --package=github:jzubielik/chief-of-staff-communication-agent#path:mcp chief-of-staff-mcp"
      ],
      "env": {
        "COS_API_URL": "https://klxrwe0sa3.execute-api.us-east-2.amazonaws.com",
        "COS_API_TOKEN": "<paste your per-user token from the dashboard's Settings > MCP Tokens view>"
      }
    }
  }
}
```

Each teammate must replace `COS_API_TOKEN`'s placeholder with their own token (see
`skills/use-pidgeot/reference/mcp-setup.md`) — unlike `elephant`, this server is per-user scoped,
so there is no single shared value to bundle.
